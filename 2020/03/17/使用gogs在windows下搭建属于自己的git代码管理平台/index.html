使用gogs在windows下搭建属于自己的git代码管理平台

发表于 2020-03-17 | 分类于 git
字数统计：473 字 | 阅读时间 ≈ 1 分钟
一、下载 gogs 程序

官网下载地址：https://dl.gogs.io/。

下载对应自己系统的版本程序，我自己使用的是gogs_0.11.91_windows_amd64.zip，下载完毕后，解压到自己想解压的磁盘目录中，我是解压到D盘的。

二、安装 git windows版本

下载地址：https://git-scm.com/downloads

三、下载NSSM

下载地址：https://nssm.cc/download

用于注册windows服务，将gogs注册到windows的服务。

四、配置环境变量
  1. 配置 git windows环境变量：D:\Git\cmd 和 D:\Git\usr\bin 。
  2. 配置 NSSM 目录：D:\nssm-2.24\win64
五，配置 gogs

找到解压后的 gogs 目录下的 script/windows 文件夹，里面有一个 install-as-service.bat 的脚本，如果 gogs 不是解压在C盘中，目录为：C:/gogs ，需要右键编辑这个脚本文件，将 SET gogspath=C:\gogs 改成自己解压gogs的目录。

六、初始化gogs

浏览器打开: http://localhost:3000，如果能正常打开，说明所有的服务没有问题，已经安装成功，接下来就可以配置gogs的属性。

数据库类型：SQlite3（如果没安装mysql可直接使用sqlite3）

数据库文件路径：默认

仓库目录：自己定义修改成自己需要放的路径

运行系统用户：cofface（自己电脑用户名）

域名：127.0.0.1

SSH端口号：10022

HTTP端口号：3000

应用URL：http://127.0.01:3000

以上的配置项，配置后基本就可以了，如果需要修改，可以自行修改 gogs 的配置文件。

配置文件在解压的gogs目录中：D:\gogs\custom\conf\app.ini

问题
  1. 如果SSH端口无法连接，需要修改 app.ini 中的 START_SSH_SERVER = true。修改了配置文件，需要重启服务，运行 -> services.mas 进入到服务中，找到 gogs 重启项目即可。如果还不能访问，可能是防火墙问题，试着关闭防火墙。
