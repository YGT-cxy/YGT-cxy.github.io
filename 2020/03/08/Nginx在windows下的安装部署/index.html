Nginx在windows下的安装部署

发表于 2020-03-08 | 分类于 Nginx
字数统计：2k 字 | 阅读时间 ≈ 8 分钟
一、官网下载Nginx

官网地址： http://nginx.org/en/download.html



根据不同的系统平台，选择不同的下载，带有 Windows的下载是适用于 windows 系统的，第一个是适用于 Linux 系统平台的。

二、安装部署
  1. 下载完毕后，解压缩文件包，解压出一个文件夹，里面包含了有关于nginx的所有东西。

  2. 启动nginx，启动的方式有两种，一种是通过 cmd 命令行的方式（推荐），一种是通过直接双击 nginx.exe 运行文件。

    (1) 双击 nginx.exe 启动。

         通过双击 nginx.exe 启动后，会有一个一闪而过的界面。

         查看是否启动成功过，需要打开 任务管理器 ，通过查看里面的进程，如果启动成功，会看到两个nginx的进程，如果启动失败，在进程里面是找不到的。



    (2) 通过 cmd 命令行启动。

    1、 使用命令到达nginx解压缩后的目录
    2、 启动nginx服务的命令
    start nginx
    3、 查看进程是否存在，可以通过命令行或者打开任务管理器查看
    tasklist /fi "imagename eq nginx.exe"

         ps:

         如果通过命令行或任务管理器中皆找不到nginx的进程，可能是启动报错了，需要查看nginx的 error.log 日志文件。

         在nginx的 log 文件夹里，有一个 error.log 的文件，打开可以看到启动报错的信息。

         常见的报错信息：

              (1) 端口号被占用

              (2) nginx文件夹路径包含中文

         两种启动方式的不同：

    通过双击 nginx.exe 启动的，修改配置文件后，使用命令行的方式重启或停止nginx是无效的，必须通过手动关闭任务管理器里的所有nginx进程，然后再次启动才可以。

  3. 修改配置文件


    在 conf 目录下找到 nginx.conf 文件，使用文本编辑器打开，找到 serve 节点，可以在里面修改端口号，默认站点目录等各种配置项。

    保存自己修改后的配置，命令行打开的可以使用命令进行重启nginx，手动双击打开的只能通过任务管理器关闭然后重新启动。

    校验配置文件是否正确的命令
    nginx -t -c /nginx-1.17.8/conf/nginx.conf


    重新加载配置文件并重启nginx
    nginx -s reload

    通过刚刚设置的域名及端口号即可访问，例如: http://localhost:80 ，出现欢迎页面就说明nginx部署成功了。

    1. 关闭nginx服务
      快速停止：
      nginx -s stop
      完整有序的停止：
      nginx -s quit
